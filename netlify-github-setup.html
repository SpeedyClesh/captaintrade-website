My Exact Netlify + GitHub Setup for Zero-Config Deployments

Every push to main deploys in under 30 seconds with security headers, caching rules, and branch previews. Here's the exact netlify.toml config I use on every project.

CaptainTrade
Developer · AI Engineer · Trader
February 10, 2026 Published
3 min read Read time

Why Netlify over other options

I've deployed projects on GitHub Pages, Vercel, Railway, and Render. For static sites and simple web projects, Netlify wins on the combination of speed, features, and simplicity. The free tier is genuinely useful — not crippled — and the deployment pipeline is the fastest I've used.

The key advantage for me is the netlify.toml config file. Everything about the deployment — headers, redirects, caching, build commands — lives in a file in the repo. No clicking around a dashboard, no undocumented settings, no surprises when you clone the project somewhere else.

What this setup gives you

Automatic deploys on every push to main. Branch previews for every PR. Security headers that score A+ on securityheaders.com. Asset caching that makes repeat visits near-instant. All free.

Repo structure that works

For a static site, keep it simple. The publish directory should be the root — no build step, no dist folder to manage:

your-project/
├── index.html
├── netlify.toml        # ← the key file
├── robots.txt
├── sitemap.xml
├── css/
├── js/
└── assets/

The netlify.toml config

This is the exact config I use on every static project. Copy it, adjust the cache durations for your use case, and you're done:

# Build settings
[build]
  publish = "."

# Security headers — applied to every page
[[headers]]
  for = "/*"
  [headers.values]
    X-Frame-Options = "DENY"
    X-Content-Type-Options = "nosniff"
    X-XSS-Protection = "1; mode=block"
    Referrer-Policy = "strict-origin-when-cross-origin"

# Aggressive caching for assets (1 year)
[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

# CSS and JS — 24 hour cache (update on each deploy)
[[headers]]
  for = "/css/*"
  [headers.values]
    Cache-Control = "public, max-age=86400"

[[headers]]
  for = "/js/*"
  [headers.values]
    Cache-Control = "public, max-age=86400"

# Clean URLs and SPA fallback
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

Branch previews for free

One of the most underrated Netlify features: every branch you push gets its own preview URL automatically. Create a branch called feature/new-section, push it, and Netlify generates a unique URL like deploy-preview-42--your-site.netlify.app within 30 seconds.

This means you can share a preview of any change with a client or colleague before it goes live — no local setup required on their end, just a link. For client work, this is genuinely invaluable.

Adding a custom domain

When you're ready for a custom domain, the process in Netlify is straightforward. Go to Site configuration → Domain management → Add domain. Netlify will give you nameservers to point your domain registrar at, and it handles SSL certificate provisioning automatically via Let's Encrypt.

The entire process from purchase to live custom domain with HTTPS takes about 15 minutes — mostly waiting for DNS propagation. Netlify's interface walks you through each step clearly.
