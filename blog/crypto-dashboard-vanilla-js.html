Building a Real-Time Crypto Dashboard With Vanilla JS and Zero Dependencies

No React. No npm. No build tools. Just clean JavaScript talking directly to public APIs — and it's faster than most framework-based solutions I've seen.

CaptainTrade
Developer · AI Engineer · Trader
March 28, 2026 Published
6 min read Read time

Why vanilla JS?

Every time I mention building something without a framework, someone asks why. The answer is simple: for a data dashboard that fetches from APIs and updates the DOM, a framework adds complexity without adding value. You're loading hundreds of kilobytes of library code to do something JavaScript does natively.

The dashboard I built fetches live crypto prices, updates in real-time via WebSocket, and renders portfolio calculations — all in under 12KB of JavaScript. No bundler, no node_modules folder, no build step. Open the HTML file and it works.

What we're building

A crypto dashboard that shows live prices for BTC, ETH, SOL and a custom portfolio — fetching from CoinGecko's free API and updating every 30 seconds, with WebSocket fallback for real-time prices.

Setting up the API calls

CoinGecko's public API requires no key and gives you price, 24h change, and market cap for any coin. The pattern is simple — fetch on load, then set an interval:

const COINS = ['bitcoin', 'ethereum', 'solana'];
const BASE  = 'https://api.coingecko.com/api/v3/simple/price';

async function fetchPrices() {
  const ids = COINS.join(',');
  const url = `\${BASE}?ids=\${ids}&vs_currencies=usd&include_24hr_change=true`;
  const res  = await fetch(url);
  const data = await res.json();
  renderPrices(data);
}

fetchPrices();
setInterval(fetchPrices, 30000);

Rendering live data efficiently

The key insight for DOM performance: never rebuild the entire list on each fetch. Instead, target specific elements by ID and update only the values that changed. This avoids layout thrashing and keeps the UI smooth even with frequent updates.

function renderPrices(data) {
  COINS.forEach(coin => {
    const d      = data[coin];
    const price  = d.usd.toLocaleString('en-US', { maximumFractionDigits: 2 });
    const change = d.usd_24h_change.toFixed(2);
    const isUp   = d.usd_24h_change >= 0;

    document.getElementById(`price-\${coin}`).textContent  = `$\${price}`;
    document.getElementById(`change-\${coin}`).textContent = `\${isUp ? '▲' : '▼'} \${Math.abs(change)}%`;
    document.getElementById(`change-\${coin}`).className  = isUp ? 'up' : 'down';
  });
}

Adding WebSocket for real-time updates

For traders who want prices updating by the second rather than every 30 seconds, Binance's public WebSocket stream requires no authentication and pushes live price data continuously.

const ws = new WebSocket(
  'wss://stream.binance.com:9443/stream?streams=btcusdt@ticker/ethusdt@ticker'
);

ws.onmessage = (event) => {
  const { data } = JSON.parse(event.data);
  // data.c = current price, data.P = 24h change %
  updatePrice(data.s, parseFloat(data.c), parseFloat(data.P));
};

ws.onerror = () => {
  // Fallback to REST polling if WebSocket fails
  setInterval(fetchPrices, 5000);
};

Performance tips

  • Debounce DOM updates — if WebSocket is pushing updates every 100ms, batch them and render at 60fps using requestAnimationFrame
  • Cache element references — store getElementById results in variables rather than querying on every update
  • Use CSS transitions — flash a subtle highlight on price change rather than replacing the element, which is cheaper and looks better
  • Graceful degradation — always have a REST polling fallback if WebSocket is unavailable

The full pattern

The complete dashboard — prices, portfolio calculations, 24h change, and WebSocket updates — comes in around 200 lines of vanilla JavaScript. No dependencies. No build tools. Loads in under 100ms on a slow connection.
